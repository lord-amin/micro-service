‫‫مستند نصب و پیکربندی مولفه Audit-Server ورژن 5.0.0

پیش‌نیاز

نصب مولفه‌های داخلی پیک‌آسا

‫برای نصب این برنامه نیاز دارید که ابتدا مولفه Auth-Server را نصب کنید. نسخه‌های سازگار، 5.0.0 و بالاتر می‌باشند.

‫نصب ElasticSearch

‫برای نصب این برنامه نیاز دارید که مولفه Elastic Search 6.1.1 را نصب کنید.

‫نصب JDK

‫مطمئن شوید که java8 نصب شده‌است.

$ java -version
java version "1.8.0_31"
Java(TM) SE Runtime Environment (build 1.8.0_31-b13)
Java HotSpot(TM) 64-Bit Server VM (build 25.31-b07, mixed mode)

‫آماده‌سازی دیتابیس

‫در پوشه db-script/Create-DB فایلی به نام permission.sql موجود است.
‫این فایل حاوی دسترسی‌های مورد نیاز برنامه است. این دسترسی‌ها باید در دیتابیس Auth-Server وارد شود. کافیست این فایل را به‌صورت زیر بر روی دیتابیس Auth-Server اجرا کنید.

mysql --verbose -u$DBA_USER -h $DBSERVER -p$DBA_PWD auth_server < permission.sql

‫مقادیر DBA_USER، DBA_PWD و DBSERVER را متناسب با دیتابیس Auth-Server تنظیم نمایید.

نصب برنامه

‫‫فایل نصب را از repository شرکت ‫دریافت کنید. این فایل را در پوشه /tmp/audit-server/ از حالت فشرده خارج نموده و فایل install.sh را از درون آن اجرا کنید.

پیکربندی

‫‫‫فایل application.properties که در پوشه config قرار دارد، حاوی پیکربندی‌های اصلی برنامه می‌باشد. این پیکربندی‌ها در زیر شرح داده‌شده‌اند:

‫پیکربندی‌های عمومی

server.port ‫‫پورت اجرای برنامه
server.address ‫‫‫آدرس اجرای برنامه؛ ‫‫‫در صورتی که نیاز دارید این مولفه از سرورهای دیگر قابل دسترسی باشد، باید مقدار این پارامتر را برابر 0.0.0.0 قرار دهید.
logging.level.root ‫‫‫سطح لاگ برنامه

‫پیکربندی اتصال به مولفه ElasticSearch

app.audit.elastic.ip ‫‫‫آدرس IP مولفه ElasticSearch
app.audit.elastic.port پورت مولفه ElasticSearch
app.audit.elastic.clustername ‫نام Cluster مولفه ElasticSearch
app.audit.elastic.partitioning ‫نوع partitioning مولفه ElasticSearch؛ مقادیر مجاز DAY, WEEK, MONTH هستند.

‫پیکربندی اتصال به مولفه Auth-Server

rs.security.enable ‫فعال بودن کنترل دسترسی‌ها؛ مقادیر مجاز true و false هستند.
rs.security.auth.server.context.path ‫آدرس اتصال به مولفه Auth-Server؛ این کانفیگ می‌تواند به‌صورت http://localhost:1998/auth-server/v3 باشد.

‫در‌صورتی‌که مقدار کانفیگ rs.security.enable را برابر false قرار دهید(به‌معنای عدم نیاز به کنترل دسترسی‌ها)، نیازی به نصب مولفه Auth-Server نیست. ‌هم‌چنین نیازی به تنظیم کانفیگ rs.security.auth.server.context.path نخواهیدداشت.

راه‌اندازی

‫‫اجرا و توقف برنامه از طریق سرویس انجام می‌گیرد:

/etc/init.d/pa-audit-server start
/etc/init.d/pa-audit-server stop

مشاهده لاگ

‫‫‫لاگ سیستم در شاخه log در محل نصب ذخیره می شود:

tail -f /usr/local/PeykAsa/audit-server-4.0.0/log/pa-audit-server.log
